Overview
What You Are Installing
FalconX is a two-part application — a Next.js frontend and a Python FastAPI backend — that run simultaneously on your local machine and connect to CDK internal systems.
01
🖥️ Next.js Frontend
The visual interface in your browser. Runs on localhost:3000. Handles login, dashboards, Jira views, AI chat, and all user interactions.
02
⚙️ FastAPI Backend
The engine behind the scenes. Runs on localhost:8000. Connects to Jira, Confluence, LDAP, GitHub, and the Databricks AI/RAG pipeline.
03
🚀 start.bat
A Windows batch script included in the ZIP. It automates everything — virtual environment, dependency installation, and launching both servers with a single double-click.
💡
How it works: The frontend (browser tab) talks to the backend (running locally). The backend connects to CDK's Jira, Confluence, LDAP, and AI systems. Both servers must remain running at all times for the application to function.

Step 0
Prerequisites — Complete Before Anything Else
Three tools must be installed and accessible before setup will succeed. Open Command Prompt (Win + R → type cmd → Enter) and verify each one.
🐍
Python 3.11 or higher
Required
Python powers the backend server. Version 3.11 or newer is required. Run the following in Command Prompt to verify:
Command Prompt
C:\Users\You> python --version
Python 3.11.8  ← Anything 3.11 or above is fine ✓
⚠️
If you see an error or a version below 3.11: Download the latest Python installer from python.org/downloads. During installation, tick the checkbox "Add Python to PATH" — this step is essential or the command will not be found. Restart Command Prompt after installation.
🟢
Node.js 18 LTS or higher
Required
Node.js is required to run the Next.js frontend. In the same Command Prompt window, run:
Command Prompt
C:\Users\You> node --version
v20.11.0  ← v18 or above is fine ✓
C:\Users\You> npm --version
10.2.4  ← npm ships bundled with Node.js ✓
⚠️
If Node.js is missing: Download the LTS version from nodejs.org. Run the installer with default settings. Close and reopen Command Prompt after installation, then re-run both version checks.
🔐
CDK VPN / Network Access
Required for Jira & Login
FalconX connects to internal CDK systems that are only accessible on the CDK network or via VPN. Confirm connectivity:
Command Prompt
C:\Users\You> ping projects.cdk.com
Reply from 10.x.x.x: bytes=32 time=12ms TTL=128  ← Connected ✓
⚠️
If the ping times out or says "could not find host": You are not on the CDK network. Connect to the CDK VPN before continuing. Contact IT if you do not have VPN access credentials.

Step 3
Configure Your .env Credentials File
This is the only step requiring your personal information. Complete this before running start.bat. The .env file tells the backend how to connect to Jira, LDAP, and Databricks.
A
Copy the template to create your .env file
~1 min
Navigate to backend\config\ inside your extracted project. You will see a file named env.template. Make a copy and rename it exactly .env.
Command Prompt — run from project root
# Navigate into the config folder
C:\Projects\Leadership_Engine_Dev-main> cd backend\config
# Copy the template to create .env
C:\...\backend\config> copy env.template .env
1 file(s) copied.
⚠️
Windows may append .txt to the filename. If the file appears as .env.txt in Explorer, go to View → Options → uncheck "Hide extensions for known file types", then rename to remove the .txt. The file must be named exactly .env with no other extension.
B
Open .env in VS Code or Notepad and fill in your values
~5 min
Open the .env file and fill in each value. Your CDK email and Jira API token are your own — Ramesh provides the Databricks details, LDAP service account credentials, and the shared ENCRYPTION_KEY.

Run start.bat — Launch the Application
With your .env file complete, you are ready to start. start.bat installs all dependencies and launches both servers automatically.
1
Double-click start.bat from the project root folder
Keep terminal open!
Open File Explorer, navigate to C:\Projects\Leadership_Engine_Dev-main\, and double-click start.bat. Or run it from the terminal:
Command Prompt — project root
C:\Projects\Leadership_Engine_Dev-main> start.bat
What start.bat does automatically — in order:
1
Port Check
Detects and frees ports 3000 and 8000 if another process is using them.
2
Python venv
Creates a Python virtual environment at backend\venv\ if it doesn't exist.
3
pip install
Installs all Python packages from requirements.txt into the venv.
4
npm install
Installs all Node packages into frontend\node_modules\.
5
Backend starts
Launches FastAPI server on localhost:8000 in a new terminal window.
6
Frontend starts
Builds and launches Next.js on localhost:3000. Browser opens automatically.
⏳
First-time installation takes 5–10 minutes. Python and Node packages need to download on the first run. Subsequent runs are near-instant as packages are cached. Do not close any terminal windows during this process.

Wait for both servers to report they are ready

Step 6
Common Issues & How to Fix Them
If something goes wrong, match your error to one of the cases below. Always check the logs/ folder first — the log files contain detailed error messages that pinpoint the exact cause.
❌ "python" is not recognised as a command
Python is not installed or was not added to PATH during installation. Try alternate commands first, then reinstall if needed.
# Try these alternate commands:
py --version
python3 --version
# If both fail: reinstall Python from python.org
# Tick "Add Python to PATH" during install
# Restart Command Prompt and retry
❌ Port 3000 or 8000 is already in use
Another process (a previous failed start, another Node app) is blocking that port. Find and terminate it:
# Find what is using port 8000
netstat -aon | findstr :8000
TCP 0.0.0.0:8000 LISTENING 12345
# Kill the process using that PID number
taskkill /f /pid 12345
SUCCESS: Process terminated.
# Repeat for port 3000, then re-run start.bat
❌ pip install fails or packages not found
pip is outdated or broken. Repair it, then re-run the installation:
python -m ensurepip --upgrade
python -m pip install --upgrade pip
# Then re-activate venv and reinstall
cd backend
venv\Scripts\activate
pip install -r requirements.txt
❌ npm install fails or node_modules is missing
Package lock file or cache is corrupted. Clean everything and reinstall from scratch:
cd frontend
rmdir /s /q node_modules
del package-lock.json
npm cache clean --force
npm install
❌ Cannot connect to Jira / login fails
Almost always a VPN issue or incorrect .env credentials. Check in this order:
# 1. Check VPN connectivity
ping projects.cdk.com
# Must reply — if not, connect VPN first
# 2. Check Jira status endpoint in browser
http://localhost:8000/api/jira/status
# 3. After fixing .env, restart backend
RESTART_BACKEND.bat
❌ .env changes are not taking effect
The backend only reads the .env file at startup. Restart it after every edit. Also verify the filename is correct:
# Confirm the file name is exactly ".env"
dir backend\config
.env ← must NOT show as .env.txt
# Restart backend only (frontend stays running)
RESTART_BACKEND.bat
❌ Frontend shows blank page or "Cannot connect"
Next.js may still be compiling on the first load (can take up to 60 seconds), or the backend is not running:
# Step 1: Wait 60 seconds and press F5 to refresh
# Step 2: Open DevTools (F12) to check console errors
# Step 3: Verify backend is running:
http://localhost:8000/health
# Step 4: Check the frontend log file:
type logs\frontend-dev.log
❌ AI Chat returns no response or times out
Databricks credentials in .env are missing or incorrect. Check the RAG health endpoint first:
# Open in browser to see exact error
http://localhost:8000/api/rag/health
# Verify these three .env values with Ramesh:
OPENAI_API_KEY ← Databricks token
OPENAI_API_ENDPOINT ← Databricks serving URL
DATABRICKS_WORKSPACE_URL ← workspace base URL
