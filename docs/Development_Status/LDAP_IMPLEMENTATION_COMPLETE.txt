🔄 AUTHENTICATION FLOW
═════════════════════════════════════════════════════════════════════════════

1. User enters CDK username/password
   ↓
2. Frontend → POST /api/auth/login
   ↓
3. Backend authenticates via LDAP
   ├─ Connects to LDAP server
   ├─ Searches for user (sAMAccountName)
   └─ Validates password
   ↓
4. Returns JWT token if successful
   ↓
5. Frontend → POST /api/auth/auto-connect-integrations
   ↓
6. Backend decrypts service credentials
   ├─ Connects to Jira (rameshaj)
   └─ Connects to Confluence (rameshaj)
   ↓
7. Returns connection status
   ↓
8. User redirected to main app
   └─ Jira/Confluence already connected!
