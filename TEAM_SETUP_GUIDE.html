Installation Steps
1
Get the Code ~1 min

Clone the repository and navigate into the project folder.

# Clone the repository (get the URL from Ramesh) git clone <repository-url> # Navigate into the project cd Leadership_Engine_Dev-main
💡 If you already have the ZIP file, extract it and skip the git clone step. Just open the extracted folder.
2
Windows Quick Start (Recommended) ~8 min

The included script handles everything automatically — virtual env, backend deps, frontend deps, and starts both servers.

# From the project root, double-click OR run in terminal: start.bat # What it does automatically: # ✓ Creates Python virtual environment (backend/venv) # ✓ pip install -r requirements.txt # ✓ npm install (frontend) # ✓ Starts frontend on http://localhost:3000 # ✓ Starts backend on http://localhost:8000
✅ Let this run fully. You'll see "Backend starting at http://localhost:8000" when it's ready.
⚠️ Do NOT close the terminal window — it keeps both servers running.
2
alt
Manual Setup (Linux / Mac / Fallback) ~10 min

Use two separate terminals. Run these in order:

──── Terminal 1: Backend ──────────────────────────── cd backend python -m venv venv # Windows: venv\Scripts\activate # Mac/Linux: source venv/bin/activate pip install -r requirements.txt python run_server.py
──── Terminal 2: Frontend ──────────────────────────── cd frontend npm install npm run dev
3
Configure Environment Variables ~5 min

The backend needs a .env file with credentials. Create it in backend/config/

# Navigate to config directory cd backend\config # Windows — copy the template: copy env.template .env # Then open .env in Notepad / VS Code and fill in the values below
⚠️ NEVER commit the .env file to Git. It contains sensitive credentials.
4
Encrypt Your Passwords ~2 min

Jira, Confluence, and LDAP passwords must be encrypted before putting them in .env. Run the encryption script:

# From backend directory, with venv activated: cd backend venv\Scripts\activate python scripts/generate_env.py # Enter your plain-text password when prompted # Copy the encrypted output → paste into .env
💡 Ask Ramesh for the shared ENCRYPTION_KEY — all team members must use the same key so the backend can decrypt credentials.
5
(Optional) Frontend Environment ~1 min

Only needed if your backend is on a different machine or port.

# Create frontend/.env.local # Only set this if backend is NOT on localhost:8000 NEXT_PUBLIC_API_URL=http://localhost:8000
✅ If you're running both locally, you can skip this — the default is already localhost:8000.
6
Start / Restart After Config ~1 min

After setting up .env, restart the backend so it picks up the new credentials.

# Option A — Windows, restart backend only: RESTART_BACKEND.bat # Option B — Kill everything and start fresh: # Close the start.bat terminal, then re-run: start.bat

Common Issues & Quick Fixes
❌ Python not found / wrong version
Python 3.11+ is required. Try py -3 --version or python3 --version.
pip install --upgrade pip
python -m pip install -r requirements.txt
❌ Port 8000 or 3000 already in use
Kill the process using the port, or start.bat does this automatically.
# Find and kill process on port 8000 (Windows)
netstat -aon | findstr :8000
taskkill /f /pid <PID>
❌ Module not found (Python)
Virtual environment not activated, or install failed. Run:
cd backend
venv\Scripts\activate
pip install -r requirements.txt
❌ npm install fails / node_modules error
Clear the cache and reinstall from scratch:
cd frontend
rmdir /s /q node_modules
del package-lock.json
npm install
❌ Cannot connect to Jira / LDAP
Almost always a VPN or credential issue. Check:
# 1. Are you on CDK VPN?
ping projects.cdk.com

# 2. Is your .env correct?
# Check JIRA_SERVICE_EMAIL and TOKEN
# Re-run generate_env.py if unsure
❌ .env file not loading
File must be exactly named .env (not .env.txt) and placed in backend/config/.
# Verify file exists:
dir backend\config\.env

# Restart backend after any .env change!
RESTART_BACKEND.bat
❌ AI chat not responding
Check Databricks config. Hit the RAG health endpoint first:
# Open in browser:
http://localhost:8000/api/rag/health

# Check OPENAI_API_KEY and
# DATABRICKS_WORKSPACE_URL in .env
❌ Frontend shows blank page
Next.js may still be building. Wait 30s, then check browser console (F12):
# Check frontend build log:
logs\frontend-dev.log

# Or restart frontend:
cd frontend
npm run dev
